# Orbit Explorer — Classroom Build

An interactive visualization tool for exploring parameterized sequence rules through graph-based pattern hunting. This is the actively-developed *classroom* build, tuned for use in front of a class (on a projector) and with middle-school / introductory audiences.

**Launch:** <https://ohiomathteacher.github.io/orbit-explorer-classroom/>

> The frozen version used in the Sichuan University study and cited in the TEA Collatz chapter — which includes the embedded "ZhengGPT" AI thinking partner — lives separately at <https://github.com/OhioMathTeacher/orbit-explorer> (<https://ohiomathteacher.github.io/orbit-explorer/>) and is intentionally not changed.

## The rule

The explorer visualizes sequences generated by a function pair:

- **f(n) = n / d** when n is even
- **f(n) = m·n + b** when n is odd

where **m**, **b**, and **d** are adjustable with the sliders. Each starting value you enter is iterated under this rule; the trajectory is drawn as a graph of connected number-nodes, and the outcome (converges to 1, enters a cycle, or diverges) is detected automatically.

## How to use

1. Set **m**, **b**, and **d** with the sliders in the top bar.
2. Type a starting value in the seed box and press Enter, or click **Show steps**.
3. Watch the graph build the trajectory. Use **max steps** to control how many steps are drawn (raise it to see cycles repeat).
4. Click any number in the **Data Reference Grids** panel to seed it directly; tested values stay highlighted there.
5. Open **Cases** to compare Recent Results and Saved Cases; **Download Cases** exports a CSV.
6. Drag number-balls to pin them; **Reset Layout** releases pins; **Reset** clears the canvas. **Show Cycles** arranges cycle nodes in circular groups. **Grid** and **Light Theme** are display toggles. A **View** menu (cycle layouts, step numbers, parity coloring, zoom controls) is being added.

## For educators

Designed for mathematical inquiry — students discover patterns in sequence behavior, form and test conjectures, and compare different parameter sets. Language support: English, Spanish, Simplified Chinese.

## Technical details

- Single HTML file; React (18) loaded from a CDN, JSX transpiled in-browser via Babel-standalone. No build step, no server.
- Canvas-based force-directed graph rendering.
- Runs entirely in your browser; nothing is sent anywhere.
